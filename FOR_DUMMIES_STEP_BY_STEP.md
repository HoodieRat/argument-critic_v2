# Argument Critic: For Dummies Quick Example

This is the fastest way to see the new upgrades.

## 1) Start the app

- Double-click Start Argument Critic.cmd
- Wait until the app window appears

## 2) Make a fresh session

- Click New Session
- Set mode to Critic

## 3) Paste this test argument

Remote work is always better than office work because commuting is wasted time, home is quieter, and productivity always increases when people are alone. Teams can still collaborate online so there is no downside.

## 4) Send it

- Click Send
- Wait a few seconds for response + analysis

## 5) Open the upgrades

- In the right-side panel, click Analysis

The desktop window should expand to a maximized analysis workspace while Analysis is open.

When you leave Analysis and return to Questions or another main area, the window should return to its earlier drawer size and position.

You now get one large Analysis Workspace in a single view (not tucked into tabs).

In that one screen, you should see:
- A clear top summary showing the highest-impact fix and the active lens
- Root claims
- Core assumptions
- Top weak spots (epistemic map)
- Issue breakdown (bar chart style summary)
- How different lenses read the argument (fit comparison + active lens detail)
- Perspective library (plain-language form for adding perspectives)

If you click Use this lens in the fit section, the selected card is highlighted, the top banner updates to Active lens: <name>, and you get a quick confirmation message.

In the Perspective library, the secondary action is labeled Set as active lens.

## 6) See another upgrade

- Click Questions
- You should now see critique-type labels on follow-up questions

## 7) If you still see nothing

- Make sure mode is Critic
- Send a full paragraph, not a one-line sentence
- Try a brand-new session and send again

That is it. This flow should make the upgrades visible in under 2 minutes.
